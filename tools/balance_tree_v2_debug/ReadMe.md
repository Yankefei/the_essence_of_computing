本文件中B树的版本为国内教科书作者严蔚敏提供的版本，和文件夹 balance_tree 中定义的B树版本有所不同。

该版本的特点为：（B树的阶为M）
1. 树的每个节点最多有M棵子树, 也就是每个节点中的元素最多有M-1个。
2. 根节点如果不是叶子节点的话，那么最少可以有2棵子树，也就是只有一个元素。
3. 除了根之外，所有节点(包含叶子节点)最少有M/2（向上取整）棵子树, 也就是除了根节点，所有其他节点
   的元素数最少为M/2（向上取整）-1 个节点。
4. 所有的叶子节点的深度都相同。
5. 所有节点均有序排列。

通过上面B树特征的分析，M最小是3，如果存在一个2阶B树，里面只有2个元素的话，那么将不满足特点2中根有
2棵子树的要求，与定义相矛盾。
（ balance_tree 文件夹中定义的B树版本最小阶可以是2）

下面是3棵3阶B树的示意图：

本文件夹 balance_tree_v2_debug 内，实现了该B树4个版本的代码，分别是

1. 没有parent_指针的递归插入，删除版本
2. 没有parent_指针的非递归插入，删除版本
3. 携带parent_指针的递归插入，删除版本
4. 携带parent_指针的非递归插入，删除版本

版本说明：

    balance_tree_v2_debug 文件夹中实现严蔚敏等国内教材B树所有的版本的算法代码，但为了便于调试，内部数据结构
定义没有采用最优方式，而且使用了一种 gdb下查看内存更直观的方式。所以文件夹后缀携带有 `_debug`. 非debug的版本
暂时没有提供，因两者的算法完全一致，如果读者仅参考算法代码，该版本已经可以提供所有信息， 非debug的版本实现仅需
要将Node节点中的普通Entry数组修改为存储Entry*指针类的指针数组，仅仅做代码迁移即可，待后面时间充裕时完善。

实际使用中建议使用非debug的版本，因为非debug和其他版本B树，B+树一样有下列的优点，读者也可以自行完成该版本的代码：

1. 在插入，删除过程中，如果需要在Node节点内以及Node节点间转移Entry数据时，转移指针是最优选择，如果
   使用普通的Entry数组，则需要额外的对象赋值操作，这个是主要考虑的点。
2. 使用指针数组的形式，可以在模板类型T本身内存较大时，去掉一次性大块内存数组申请的需要。
